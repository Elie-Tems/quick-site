# אפיון: וריאציות מוצר (צבעים / מידות / מלאי) - לחנויות בגדים

> אפיון + עיצוב לאישור. לא נגעתי בקוד.

## המלצה: זה **לא** מודול נפרד - זו יכולת בתוך מודול "חנות מוצרים"

מוטי שאל "נעשה את זה כמודול גם, נכון?". דעתי - **לא כמודול נפרד, אלא כתכונה של המוצר**, מהסיבות האלה:

- **מודול** = משנה מה האתר *הוא* (חנות מול יומן מול תרומות). וריאציות הן **מאפיין של מוצר בתוך החנות** - אי אפשר "וריאציות" בלי "מוצרים".
- כל חנות בגדים/נעליים/אביזרים צריכה את זה - זה חלק מ"למכור מוצרים כמו שצריך", לא יכולת אופציונלית נפרדת.
- מודול נפרד היה יוצר בלבול (מדליקים "וריאציות" בלי "מוצרים"?).

לכן: **כפתור/אזור "וריאציות" בתוך עורך המוצר**, זמין לכל חנות מוצרים. אפשר בעתיד לתמחר אותו כשדרוג (כמו CRM) - אבל מבחינת ארכיטקטורה זה שייך למודול המוצרים.

---

## מודל הנתונים

טבלה חדשה `product_variants` (וריאנט = צירוף ספציפי של מאפיינים):

| עמודה | סוג | תיאור |
|---|---|---|
| `id` | uuid | מזהה |
| `product_id` | uuid → products | המוצר |
| `color` | text | צבע (למשל "שחור") - null אם אין מאפיין צבע |
| `color_hex` | text | קוד צבע להצגת עיגול (#000000) |
| `size` | text | מידה (S/M/L/38/40...) - null אם אין |
| `sku` | text | מק"ט אופציונלי |
| `stock` | integer | כמות במלאי לצירוף הזה |
| `price_override` | numeric | מחיר שונה לוריאנט (אופציונלי - null = מחיר המוצר) |
| `image_url` | text | תמונה ספציפית לצבע (אופציונלי) |

**למה שורה-לכל-צירוף (ולא שתי טבלאות נפרדות):** צבע שחור במידה M יכול להיות במלאי, ושחור ב-L אזל - צריך מלאי **לכל צירוף**. שורה אחת = צירוף אחד = מלאי אחד. פשוט ומדויק.

מוצר בלי וריאציות = אפס שורות ב-`product_variants`, ומתנהג בדיוק כמו היום (המבנה הקיים לא משתנה - תאימות לאחור מלאה).

## מוקאפ א' - עורך המוצר (צד הסוחר)

```
┌─ עריכת מוצר: חולצת טי ─────────────────────────────────┐
│  שם, מחיר, תמונות, תיאור...            (כמו היום)        │
│                                                          │
│  ▸ וריאציות (צבעים ומידות)              [+ הפעל וריאציות]│
│  ┌──────────────────────────────────────────────────┐  │
│  │  צבעים:  ● שחור   ● לבן   ● כחול   [+ הוסף צבע]     │  │
│  │  מידות:  S   M   L   XL              [+ הוסף מידה] │  │
│  │                                                    │  │
│  │  מלאי לכל צירוף:                                    │  │
│  │            S     M     L     XL                    │  │
│  │  ● שחור  [12]  [ 8]  [ 0]  [ 5]                    │  │
│  │  ● לבן   [ 4]  [15]  [ 7]  [ 2]                    │  │
│  │  ● כחול  [ 0]  [ 3]  [ 9]  [ 0]                    │  │
│  │                                                    │  │
│  │  (0 = אזל; יוצג ללקוח כ"אזל מהמלאי")               │  │
│  └──────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

- הסוחר מגדיר **רשימת צבעים** (עם בורר צבע לעיגול) ו**רשימת מידות** (טקסט חופשי - S/M/L או 38/40).
- המערכת בונה **מטריצת מלאי** (צבע × מידה); ממלאים כמות לכל תא. תא = 0 → "אזל".
- מחיר/תמונה שונים לצבע = אופציונלי (הרחבה, לא חובה בשלב 1).
- מוצר בלי וריאציות - פשוט לא מפעילים את האזור.

## מוקאפ ב' - דף המוצר (צד הלקוח)

```
┌─ חולצת טי · ₪89 ──────────────────────────┐
│  [ תמונה ]                                  │
│                                             │
│  צבע:   ● שחור   ○ לבן   ○ כחול             │
│  מידה:  [S] [M] [L̶]̶ [XL]     (L מסומן אזל)  │
│                                             │
│  ✓ במלאי (8 יחידות)                         │
│  [ הוסף לעגלה ]                             │
└─────────────────────────────────────────────┘
```

- הלקוח בוחר **צבע** (עיגולי צבע) ו**מידה** (כפתורים).
- צירוף שאזל - **מידה מושבתת/מסומנת** ("אזל"), לא ניתן להוסיף לעגלה.
- הכמות במלאי לצירוף הנבחר מוצגת; ההזמנה **מפחיתה מהמלאי** של אותו צירוף.
- בעגלה/הזמנה נשמר הצבע+המידה שנבחרו (כדי שהסוחר יידע מה לארוז).

---

## מה צריך לבנות (היקף)

1. **מיגרציה:** טבלת `product_variants` + RLS (owner דרך `businesses`→`profiles`, כמו שתיקנו היום) + הפחתת מלאי אטומית בהזמנה.
2. **עורך המוצר:** אזור וריאציות - ניהול צבעים/מידות + מטריצת מלאי.
3. **דף המוצר בחנות:** בוחר צבע+מידה, מצב "אזל", והוספה לעגלה עם הצירוף.
4. **הזמנה:** לשמור variant_id בשורת ההזמנה + להפחית מלאי בעת הרכישה.
5. **CRM/רווחיות:** המלאי לכל צירוף מוצג בדשבורד (אופציונלי בשלב 2).

## הצעת שלבים

- **שלב 1:** מטריצת צבע×מידה + מלאי + "אזל" בחנות (הליבה - זה מה שביקשת).
- **שלב 2:** מחיר/תמונה שונים לצבע, מק"ט, והתראת "מלאי נמוך" בדשבורד.
