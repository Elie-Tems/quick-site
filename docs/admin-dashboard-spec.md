# אפיון: דשבורד העל (אדמין) כמרכז שליטה מבוסס-CRM

מסמך אפיון לארגון מחדש של **דשבורד העל של סיאנגו** (`/manage-x7k9`) - המערכת הפנימית שבה מוטי והצוות מנהלים את כל הסוחרים בפלטפורמה. המטרה: להפוך אותו ממערכת עשירה-אך-מפוזרת (8 קבוצות, ~30 מסכים) ל-**מרכז שליטה שבו הסוחר הוא הלקוח** (Merchant CRM), עם אזור **הכנסות ודוחות** מאוחד - בלי להוריד אף יכולת קיימת.

מבוסס על מיפוי מלא של האדמין הקיים.

---

## 1. החזון

באדמין, **הסוחרים הם הלקוחות שלנו.** היום המידע על כל סוחר מפוזר: רשימת לקוחות במסך אחד, רשימת חנויות באחר, חיובים בשלישי, רדומים ברביעי, נטישה בחמישי. אין מקום אחד שמראה "מי הסוחר הזה, כמה הוא שווה לנו, מה מצב הבריאות שלו, ומה לעשות איתו".

המעבר: הדשבורד נבנה מחדש סביב **CRM של סוחרים** - כרטיס-על 360 לכל סוחר (זהות + חנות + MRR + חיובים + בריאות + סיכון נטישה + ציר זמן + פעולות) - ואזור **הכנסות ודוחות** מאוחד עם מקור-אמת אחד.

**3 עקרונות:**
1. **שום יכולת לא נמחקת.** כל ~30 המסכים ממשיכים לעבוד - רק מתארגנים ומתחברים. טבלת מיפוי (סעיף 7).
2. **מקור-אמת אחד.** היום MRR מחושב ב-3 מקומות שונים עם לוגיקה שונה. מאחדים.
3. **סוגרים פערים קיימים.** קוד מת (`AdminUsersList` בנוי ולא מחובר), הגדרות ריקות, אין CRM-סוחר מאוחד, ייצוא מפוזר.

---

## 2. הבעיה היום

מהמיפוי המלא עלו הפערים הבאים:
- **אין CRM-סוחר מאוחד.** `AdminCustomers` (רשימת אנשים), `AdminBusinesses` (רשימת חנויות), `AdminDormant` (רדומים), וחיובים פר-לקוח - כולם חתיכות נפרדות שלא מחוברות לכרטיס אחד.
- **קוד מת + כפילות.** `AdminUsersList` בנוי במלואו אבל **לא מחובר** לשום מקום; חופף ל-`AdminCustomers`. שתי תצוגות "אנשים" מקבילות.
- **MRR ב-3 מקומות** (Command Center, Overview, מסך MRR) עם 3 חישובים שונים ו-fallback קשיח ₪69. אין מקור-אמת.
- **אין אזור דוחות מאוחד.** ייצוא CSV מפוזר פר-מסך; מסכי האנליטיקס קריאה-בלבד בלי מרכז ייצוא/דוחות.
- **הגדרות = placeholder ריק.** אין ניהול feature-flags, אין ניהול הרשאות/אדמינים, אין אכיפת MFA.
- **3 משפכים נפרדים** (הרשמה / קונים / UTM) בלי קישור ביניהם.
- **תקרות סקייל** (300 חנויות, 100 הזמנות, בלי pagination בצד-שרת).

---

## 3. המבנה החדש - 7 אזורים

```
1. מרכז שליטה           - דופק, KPI, התראות חכמות, פעילות חיה
2. סוחרים · CRM          ⭐ הליבה
     • כל הסוחרים (directory: חיפוש, פילטרים, בריאות, שלב מחזור-חיים)
     • כרטיס סוחר 360 (זהות + חנות + MRR + חיובים + בריאות + סיכון נטישה + ציר זמן + הערות/פעולות)
     • בסיכון / רדומים (churn + dormant מאוחדים)
     • הזמנות סוחרים
3. הכנסות ודוחות         ⭐ מאוחד
     • MRR / ARR (מקור-אמת אחד)
     • Funnel · נטישה · Cohort retention
     • הכנסות סיאנגו מול GMV סוחרים
     • Marketplace · Top performers · קטגוריות · צפיות
     • דוחות וייצוא (מרכז ייצוא אחד)
4. תשלומים               - תשלומים · ביטולים · שגיאות תשלום · החזרים
5. תמחור ושותפים         - דומיינים · מייל · קופוני מנוי · רווחי שותפים · שיווק (רכישת לקוחות של סיאנגו)
6. תקשורת ודיוור         - וואטסאפ · הבוט שלנו · רשימת הסרות · לוג מיילים
7. מערכת                 ⭐ (נבנה מה-placeholder) - הגדרות · feature-flags · הרשאות ואדמינים · MFA
```

**השינויים המרכזיים מול היום:**
- **"סוחרים · CRM" נולד** - מאחד את AdminCustomers + AdminBusinesses + הקוד-המת AdminUsersList + AdminDormant + החיובים-פר-לקוח לכרטיס-על אחד עם ציון בריאות ושלב מחזור-חיים.
- **"הכנסות ודוחות" מאוחד** - מאחד את קבוצות "הכנסות" + "תובנות ושוק" הנפרדות; מקור-אמת אחד ל-MRR; מרכז ייצוא.
- **"מערכת" נבנה** מה-placeholder הריק - feature-flags, הרשאות, MFA.

---

## 4. סוחרים · CRM (הליבה)

### 4.1 Directory - כל הסוחרים
טבלה/כרטיסים אחודים של כל סוחר, עם עמודות שהיום מפוזרות: שם, מייל, טלפון, חנות, קטגוריה, תוכנית, פורסם, תאריך הצטרפות, **MRR פר-סוחר**, **שלב מחזור-חיים** (חדש / פעיל / בסיכון / רדום / נוטש), **ציון בריאות**. חיפוש + פילטרים + pagination בצד-שרת (סוגר את תקרת ה-300).

### 4.2 כרטיס סוחר 360 - ⭐ חדש
לחיצה על סוחר פותחת כרטיס-על אחד שמרכז את מה שהיום מפוזר ב-5 מסכים:
- **זהות + חנות** (מ-AdminCustomers/AdminBusinesses).
- **MRR + היסטוריית חיובים** (BillingChargesPanel הקיים) + **החזר**.
- **ציון בריאות ושלב מחזור-חיים** (פעילות, הזמנות אחרונות, סטטוס תשלום).
- **סיכון נטישה** (מ-churn/dormant).
- **ציר זמן פעילות** (הזמנות, פרסום, תשלומים, פניות).
- **פעולות אדמין**: מייל, צפייה בחנות, איפוס onboarding, השהיה/מחיקה, החזר, **הערות פנימיות** (חדש - CRM אמיתי).

### 4.3 בסיכון / רדומים
מאחד `AdminChurnRate` + `AdminDormant` + לקוחות "בסיכון" לרשימת פעולה אחת (מי לתפוס לפני שנוטש).

### 4.4 הזמנות סוחרים
`AdminOrdersList` הקיים (כל ההזמנות בפלטפורמה) - עובר לכאן, עם pagination.

---

## 5. הכנסות ודוחות (מאוחד)

מאחד את קבוצות "הכנסות" ו"תובנות ושוק" הנפרדות למקום אחד:

| תת-מסך | מקור | סטטוס |
|---|---|---|
| **MRR / ARR** | `AdminMRR` - **מקור-אמת אחד** (מבטל 3 החישובים) | קיים, מאוחד |
| **Funnel · נטישה · Cohort** | AdminFunnel / ChurnRate / CohortRetention | קיים |
| **הכנסות סיאנגו מול GMV סוחרים** | AdminSiangoRevenue מול AdminMarketplace/Payments | קיים, מוצג צד-לצד |
| **Top performers · קטגוריות · צפיות** | AdminTopPerformers / CategoryMap / Analytics | קיים |
| **דוחות וייצוא** | מרכז ייצוא אחד (CSV/PDF) לכל הדוחות | חדש (מחליף ייצוא מפוזר) |

**מקור-אמת ל-MRR** הוא התיקון החשוב: היום MRR מחושב ב-3 מקומות עם תוצאות שונות. מאחדים ל-hook אחד (`useMRR`) שכולם צורכים.

---

## 6. מערכת (נבנה מה-placeholder)

האזור "הגדרות" הריק הופך ל-**מערכת** אמיתי:
- **Feature-flags** - הפעלה/כיבוי של מודולים (whatsapp, email, crm...) שכבר מוזכרים בקופונים.
- **הרשאות ואדמינים** - ניהול תפקידי `admin` (RPC `has_role` קיים), הוספה/הסרה.
- **MFA** - אכיפת הרשמה ל-2FA (היום נאכף רק אם כבר נרשם).
- **קונפיג מערכת** - ערכים גלובליים במקום fallback-ים קשיחים בקוד.

## 7. טבלת מיפוי - לאן עובר כל מסך אדמין קיים

| מסך אדמין קיים | היום (קבוצה) | מיקום חדש |
|---|---|---|
| סקירה כללית / Command Center | ראשי | **מרכז שליטה** |
| פעילות חיה | ראשי | **מרכז שליטה ← פעילות** |
| לקוחות (AdminCustomers) | לקוחות וחנויות | **סוחרים · CRM ← directory + כרטיס 360** |
| חנויות (AdminBusinesses) | לקוחות וחנויות | **סוחרים · CRM** (מתמזג לכרטיס 360) |
| (AdminUsersList - קוד מת) | לא מחובר | **סוחרים · CRM** (מתמזג, מפסיק כפילות) |
| רדומים (AdminDormant) | תובנות ושוק | **סוחרים · CRM ← בסיכון/רדומים** |
| הזמנות (AdminOrdersList) | הזמנות ותשלומים | **סוחרים · CRM ← הזמנות** |
| תשלומים · ביטולים · שגיאות תשלום | הזמנות ותשלומים | **תשלומים** |
| MRR/ARR · Funnel · נטישה · Cohort | הכנסות | **הכנסות ודוחות** (MRR = מקור-אמת אחד) |
| צפיות · Marketplace · Top · קטגוריות | תובנות ושוק | **הכנסות ודוחות** |
| הכנסות סיאנגו (SiangoRevenue) | בתוך תשלומים | **הכנסות ודוחות** (צד-לצד עם GMV) |
| דומיינים · מייל · קופונים · שותפים · שיווק | תמחור ושותפים | **תמחור ושותפים** (נשאר) |
| וואטסאפ · בוט · הסרות · לוג מיילים | תקשורת ודיוור | **תקשורת ודיוור** (נשאר) |
| הפניות (Referrals) | הזמנות ותשלומים | **תמחור ושותפים ← הפניות** |
| הגדרות (placeholder) | הגדרות | **מערכת** (נבנה: flags/הרשאות/MFA) |

**כל המסכים ממופים. אף יכולת לא נמחקת - קוד מת מחובר, כפילויות מתמזגות, מדדים מתאחדים.**

## 8. תוכנית מדורגת

- **שלב 0 - שלד ניווט.** מבנה 7-האזורים החדש + מיפוי כל view. סיכון נמוך.
- **שלב 1 - סוחרים · CRM.** מאחדים לקוחות+חנויות+UsersList+רדומים לכרטיס 360; מוסיפים ציון בריאות + שלב מחזור-חיים + הערות; pagination בצד-שרת.
- **שלב 2 - הכנסות ודוחות.** מאחדים את שתי קבוצות האנליטיקס; מקור-אמת אחד ל-MRR; מרכז ייצוא.
- **שלב 3 - מערכת.** בונים feature-flags + הרשאות + MFA.
- **שלב 4 - ליטוש.** מבטלים fallback-ים קשיחים; מחברים קופונים ל-checkout (שכבר סומן "next step").

## 9. מוקאפים לאישור

מוצגים בצ'אט:
1. שלד האדמין + תפריט 7-האזורים + מרכז שליטה (דופק + התראות).
2. סוחרים · CRM - directory + כרטיס סוחר 360 (בריאות · MRR · סיכון נטישה).
3. הכנסות ודוחות - סקירה מאוחדת (MRR · נטישה · סיאנגו מול GMV).

לאחר אישור - בונים לפי התוכנית המדורגת.
