# תוכנית SEO רב-לשוני לאתר הפלטפורמה - Siango

**תאריך:** 2026-07-07
**מטרה:** שאתר הפלטפורמה (siango.app) ידורג אורגנית בכמה שוקים, כל אחד בשפתו - ושמי שמחפש יגיע לדף הבית בשפה הנכונה.

> זה נוגע ל**אתר השיווקי של Siango** בלבד. חנויות הלקוחות (`/store/...`) הן ורטיקל נפרד שכבר מטופל טכנית.

## החלטות שנסגרו עם מוטי
- **שפות (5, כולן בסקופ):** עברית (בסיס), אנגלית, ערבית, צרפתית, רוסית. RTL: he, ar. LTR: en, fr, ru. התשתית שלא-תלויה-בשפה מאפשרת להוסיף עוד בקלות.
- **מבנה כתובות:** תיקיות משנה. עברית ב-root (`/`), אנגלית `/en/`, ערבית `/ar/`, צרפתית `/fr/`, רוסית `/ru/`.
- **תרגום:** טיוטת AI + הגהה אנושית של מוטי לפני עלייה. מילות מפתח מקומיות אמיתיות לכל שוק (לא תרגום מילולי).
- **מחיר (חסום):** נמצאה סתירה ב-4 מקומות (Hero 69 / PricingSection 99-199-399 / llms 99-199-299 / schema 99-299). ממתין לטבלת מחירים אמיתית ומלאה ממוטי לפני יישור כל המקורות. **בכל הטיוטות המתורגמות - להשאיר placeholder למחיר עד שיאושר.**

## שתי מטרות נפרדות

### מטרה 1 - חיזוק הדירוג בעברית (מהיר, עצמאי)
הדף כבר ממצב את עצמו רחב ("כל הכלים, לכל תחום", עם ורטיקלים: אופנה, שירותים, נדל"ן, עמותות...). אבל ה-**meta עדיין צר**: הכותרת אומרת "אתר מכירתי" בלבד. מי שמחפש את הביטויים הרחבים "בניית אתרים", "אתר לעסק", "אתרים בזול" פחות מוצא אותנו.
- **פעולה:** להרחיב title/description/H1 שיכללו גם "בניית אתרים לעסק", "אתר לעסק בזול", לצד "אתר מכירתי". (טיוטת קופי לאישור מוטי - לא לשנות חד-צדדית את מסר הליבה.)
- **באג דיוק שנמצא אגב כך:** מחיר הכניסה בדף = **69 ₪**, אבל ה-`SoftwareApplication` schema וה-`llms.txt` עדיין אומרים 99. סתירה שגוגל רואה. צריך ליישר את כולם למחיר האמיתי. **דורש אישור מוטי מה המחיר הנכון.**

## ארכיטקטורה

### 1. ספריית i18n
`react-i18next` - הסטנדרט, עובד עם ה-`react-router-dom` v6 הקיים בלי מיגרציה, תומך lazy-load של שפות. אין כרגע ספריית i18n בפרויקט; התוכן מקודד קשיח.

### 2. מבנה תרגומים - מקור אמת אחד
קובצי JSON לפי שפה (`src/locales/he.json`, `en.json`, `ar.json`) עם מפתחות סמנטיים.
**קריטי ל-SEO:** אותם קבצים חייבים להיות נגישים גם ל**שכבת ה-edge** (Cloudflare function), לא רק ל-React, כי הזחלנים צריכים את הטקסט המתורגם ב-HTML הראשוני. לכן הקופי השיווקי יישב בקובץ משותף ש-React (i18next) וגם ה-`_middleware.ts` יטענו.

### 3. ניתוב לפי שפה
- עברית: `/`, `/templates`, `/contact`... (ללא קידומת, ברירת מחדל).
- אנגלית: `/en/`, `/en/templates`...
- ערבית: `/ar/`, `/ar/templates`...
- רק **עמודי השיווק הציבוריים** מקבלים גרסאות שפה (דף בית, תבניות, מחירים/צור-קשר, עזרה, משפטי). האפליקציה עצמה (dashboard/onboarding, מאחורי התחברות) לא צריכה URL-ים לכל שפה עבור SEO - שפת ה-UI שלה היא פרויקט נפרד (Phase 4).

### 4. hreflang (הליבה של רב-לשוני נכון)
כל עמוד שיווקי מצהיר על כל החלופות + x-default, כדי שגוגל יגיש לכל משתמש את הגרסה הנכונה בלי עונש תוכן-כפול:
```html
<link rel="alternate" hreflang="he" href="https://siango.app/" />
<link rel="alternate" hreflang="en" href="https://siango.app/en/" />
<link rel="alternate" hreflang="ar" href="https://siango.app/ar/" />
<link rel="alternate" hreflang="x-default" href="https://siango.app/" />
```

### 5. רינדור לזחלנים (מה שגורם לזה באמת לדרג)
כי זה SPA - להרחיב את `functions/_middleware.ts` שיזריק לעמודי השיווק, לפי השפה שב-path: `<title>`/meta/hreflang מתורגמים + JSON-LD + **בלוק טקסט hero גלוי-לזחלן מתורגם**. בדיוק כמו שכבר עושים לחנויות. זו אותה תשתית שהדוח סימן שצריך גם ל-`/` וגם לעמודי המשנה (C5/C6).

### 6. RTL מול LTR
עברית + ערבית = RTL. אנגלית = LTR. האתר בנוי RTL-first, אז גרסה אנגלית דורשת:
- `<html lang dir>` דינמי לפי שפה.
- מעבר מ-`mr-/ml-/text-right` ל-properties לוגיים (`ms-/me-/text-start`) בקומפוננטות השיווק, אחרת האנגלית תיראה שבורה.

### 7. סיטמאפ + מתג שפה
- סיטמאפ: להוסיף רשומות `xhtml:link` עם hreflang לכל עמוד שיווקי.
- מתג שפה ב-UI + זיהוי אוטומטי (Accept-Language) עם הפניה בביקור ראשון, תמיד עם אפשרות ידנית ו-URL קנוני יציב.

## מפת דרכים (phases)

| Phase | תוכן | תוצר |
|---|---|---|
| **0** (עצמאי) | 3 התיקונים שכבר בוצעו + מטרה 1 (הרחבת קופי עברית + יישור מחיר) | דירוג עברי חזק יותר |
| **1** | תשתית i18n + ניתוב + edge-SSR + hreflang + RTL/LTR + **דף הבית** ב-3 שפות end-to-end | דף בית שמדורג ב-3 שוקים |
| **2** | שאר עמודי השיווק (תבניות, מחירים/צור-קשר, עזרה, משפטי) באותה תבנית | כיסוי שיווקי מלא ב-3 שפות |
| **3** | הרחבת שפות: צרפתית + רוסית (ואז ספרדית וכו') = הוספת קובצי תרגום | כיסוי שוקים נוסף |
| **4** (לא-SEO, אופציונלי) | שפת UI לאפליקציה (onboarding/dashboard) | חוויית לקוח רב-לשונית |

**עקרון:** Phase 1 נבנה כפרוסה אנכית מלאה על דף הבית - כדי לראות את כל המנגנון עובד (ניתוב+תרגום+זחלן+hreflang+RTL) לפני שמשכפלים לשאר העמודים והשפות.
