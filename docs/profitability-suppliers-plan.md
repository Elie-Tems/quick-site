# רווחיות וניהול ספקים - אפיון (לאישור מוטי)

מטרה: לתת לסוחר לדעת **כמה הוא באמת מרוויח** (לא רק כמה מכר), ולנהל את הספקים שלו -
בלי לסבך את מי שלא צריך את זה.

עיקרון-על: **הכל אופציונלי.** סוחר שלא ממלא עלות/ספק - המערכת עובדת בדיוק כמו היום.
מי שכן ממלא - מקבל שכבת רווחיות שלמה. נתוני העלות/ספק הם **פנימיים בלבד** - לעולם לא
מוצגים ללקוח הקצה.

---

## 1. שדות חדשים במוצר (אופציונליים)
בטופס הוספת/עריכת מוצר, תחת אזור מתקפל "רווחיות וספק (לא חובה)":
- **מחיר עלות** (כמה המוצר עלה לכם) - מספר.
- **ספק** - שם הספק (טקסט, עם השלמה אוטומטית מספקים קיימים).

## 2. DB
- `products`: עמודות חדשות `cost_price numeric`, `supplier text`.
- `order_items`: עמודה `cost_at_order numeric` - **צילום של העלות בזמן ההזמנה**, כדי
  שדוח רווח היסטורי יישאר מדויק גם אם העלות תשתנה בעתיד. (orders-create ימלא אותה.)

## 3. מה הסוחר מקבל
**בכרטיס המוצר / רשימת מוצרים:**
- רווח ליחידה = מחיר מכירה − עלות, ואחוז רווח (margin %).

**דוח רווחיות (מסך חדש תחת "ניהול מכירות" או טאב ב"הזמנות"):**
- סך הכנסות · סך עלויות · **סך רווח** · אחוז רווח כולל - לתקופה נבחרת.
- **מוצרים הכי רווחיים** (ולא רק הכי נמכרים - לפעמים שונה!).
- **מוצרים ברווח נמוך / הפסד** - התראה כדי לתמחר מחדש.

**דוח ספקים (תת-מסך):**
- כמה הוצאתם מול כל ספק, כמה מוצרים מכל ספק, ואיזה ספק מייצר הכי הרבה רווח.

## 4. למה זה חזק
- הסוחר עובר מ"כמה מכרתי" ל"כמה הרווחתי" - זו קפיצת מדרגה בתחושת "מערכת רצינית".
- בסיס מצוין למודול בתשלום (כמו ה-CRM).
- ניהול ספקים = ערך לעסקים שקונים ממספר ספקים (רוב חנויות המסחר).

## 5. היקף בנייה (כשיאושר)
1. migration: 2 עמודות ב-products + 1 ב-order_items.
2. טופס מוצר: אזור מתקפל "רווחיות וספק".
3. orders-create: צילום cost_at_order מתוך המוצר.
4. מסך "רווחיות" + תת-מסך "ספקים" עם הדוחות.
5. תיעוד במרכז הידע + הבוט.
