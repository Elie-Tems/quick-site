# Roadmap: מסקיצות למוצר אמיתי (וורטיקלים)

מסמך עבודה. מבוסס על סריקת קוד מלאה (2026-07-07). מוטי + דניאל.
מקף רגיל בלבד, בלי em-dash.

## תמונת מצב (מה קיים בקוד)

| שכבה | מצב |
|---|---|
| מסחר (מוצרים/הזמנות/עגלה/סליקה/מנויים/חיוב/דומיינים/רווחיות/ספקים) | ✅ בוגר וחי |
| מייל-מרקטינג + וואטסאפ (backbone מלא) | ✅ בנוי, מכובה (gated) |
| CRM (`customer_crm`: תגיות/הערות + RFM) | ⚠️ קיים, נגזר מהזמנות בלבד |
| 4 פריסות חנות (Classic/Service/Property/Market) | ✅ מחווט לחנות האמיתית |
| בורר סוג-עסק (products/services/realestate/nonprofit) | 🔴 קוסמטי - לא נשמר, לא מסתעף |
| יומן/תורים · לידים/נכסים · תרומות/סעיף 46 | 🔴 0 טבלאות, 0 פונקציות. לא קיים. |

**מה הכי חסר:** (1) התשתית שהופכת סוג-עסק לאמיתי; (2) כל ה-backend של הוורטיקלים - הסקיצות הן 100% frontend.

**החלטות מוטי (2026-07-07):**
- היומן נבנה **דו-כיווני מההתחלה**. גוגל = עדיפות; Microsoft/Outlook פחות קריטי (אפשר בפיגור אם מסובך).
- **סעיף 46 אופציונלי** - לא לכל עמותה יש. חייב להיות toggle, לא הנחה.
- **המיילים תפורים כרגע לחנות רגילה.** צריך להתאים פר-וורטיקל. **כל מייל מאושר ע"י מוטי בצ'אט לפני deploy.**

## Phase 1 - התשתית (הקיסטון)
- [ ] לשמור `business_type` ל-DB + לחווט ב-useCreateBusiness.
- [ ] טבלת `business_features` (מודולים דלוקים לפי type + מנוי).
- [ ] שכבת feature-gate (טאבים/סקשנים/API לפי מודול).
- [ ] הכללת מודל: `contacts` משותף + פרטים לפי-וורטיקל + `transactions` מטופס.
- [ ] בחירת layout אוטומטית לפי business_type.

## Phase 2 - מנוע יומן/תורים (דו-כיווני מההתחלה)
- [ ] מודל: services, availability, appointments, staff.
- [ ] זרימת הזמנה בחנות (שירות → משבצת → מקדמה → אישור).
- [ ] אישור + תזכורות (reuse וואטסאפ/מייל).
- [ ] יומן ניהול לסוחר.
- [ ] **סנכרון דו-כיווני Google (קודם) + Microsoft (אחר כך):** OAuth, קריאת busy → חסימה, דחיפת תורים, webhooks, אזורי זמן, אירועים חוזרים, קונפליקטים.
- [ ] וריאנט צימרים: טווח תאריכים + מחיר ללילה + סנכרון Booking/Airbnb.

## Phase 3 - אבולוציית CRM
- [ ] ישות `contacts` עצמאית (הריפקטור המרכזי).
- [ ] מנוע שלבים/pipeline לפי וורטיקל.
- [ ] CRM לידים (נדל"ן/רכב): קונטקט + נכס-מתעניין + שלב + מעקב.
- [ ] CRM עמותה (תורמים): חד-פעמי/הוראת-קבע + סעיף 46 אופציונלי + reuse RFM/LTV.
- [ ] CRM שירותים (לקוחות): היסטוריית תורים + תזכורות.

## Phase 4 - חנויות לידים/לוחות (נדל"ן, רכב)
- [ ] סכמת listing + מדיה (תמונות/וידאו/360).
- [ ] סינון/חיפוש + מפה.
- [ ] טופס ליד במקום עגלה → CRM.
- [ ] embed 360 + דף פרויקט יזם.

## Phase 5 - מנוע תרומות
- [ ] זרימת תרומה + סעיף 46 אופציונלי.
- [ ] תרומות חוזרות (reuse מנוע החיוב iCount - כבר תומך הוראת קבע).
- [ ] קמפיינים עם יעד/התקדמות.
- [ ] גיוס המונים: מדרגות, תומכים, הכל-או-כלום.

## Phase 6 - חוצה-מערכת
- [ ] prefers-reduced-motion.
- [ ] להדליק וואטסאפ + מייל-מרקטינג (בנוי וכבוי).
- [ ] Onboarding: תצוגה חיה + נתוני דמו.
- [ ] מדידת משפך.

## מיילים - התאמה פר-וורטיקל (חוצה Phase 3/5)
המיילים היום (אישור הזמנה, ברוך-הבא, נטישה, תזכורות) תפורים לחנות מוצרים.
צריך גרסאות/התאמות ל: שירותים (אישור תור, תזכורת תור), נדל"ן (ליד התקבל), עמותה (תודה על תרומה, קבלת סעיף 46 אם רלוונטי).
**כלל: כל מייל מוצג למוטי בצ'אט לאישור לפני deploy.**
