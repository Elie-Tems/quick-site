# מחקר חברות משלוחים / API (28/6/2026)

מטרה: לאפשר לחנויות סיאנגו להציע משלוחים אמיתיים עם API קל ומתועד - בעדיפות
לחברות ישראליות, רצוי כך שה**סוחר לא יצטרך לפתוח חשבון** מול כל חברת שילוח (סיאנגו
מתממשקת פעם אחת).

## שני מודלי אינטגרציה
- **א. סיאנגו-מאסטר (מומלץ):** סיאנגו מתממשקת לספק/אגרגטור אחד ומחזיקה את החשבון;
  הסוחרים שולחים דרך סיאנגו (אנחנו מחייבים אותם, עם תוספת רווח -> הכנסה חוזרת כמו
  דומיינים/מייל). הסוחר לא צריך חשבון מול חברת שילוח. תואם ל"בלי שהוא יצטרך אותם".
- **ב. חיבור-סוחר:** כל סוחר מחבר את מפתחות ה-API שלו. יותר עבודה לסוחר, בלי רווח לנו.

## אפשרויות ישראליות (מועדף)
| ספק | מה זה | API | התאמה |
|---|---|---|---|
| **LionWheel** | פלטפורמת ניהול משלוחים ישראלית שמחברת חנויות <-> הרבה חברות שילוח (אינטגרציות eCommerce/ERP/POS, סנכרון הזמנות אוטומטי) | כן - אינטגרציות + API מתועד | **הבחירה המובילה** - בנוי בדיוק למסחר ישראלי, מאגד חברות |
| **HFD** | חברת המשלוחים B2C הגדולה בישראל, "one-stop shop" כולל נקודות איסוף/לוקרים | כן - אינטגרציה/API לחנויות | חלופה ישירה חזקה (דומיננטי בישראל) |
| **צ'יטה** | שליחויות + רשת נקודות איסוף + מעקב | יש אינטגרציות | טוב לנקודות איסוף |

## המלצה
1. **ראשי: LionWheel** (ישראלי, מאגד, בנוי לחנויות) במודל א' - אינטגרציה אחת,
   הסוחר מקבל משלוחים בלי חשבון, אנחנו מוסיפים רווח (הכנסה חוזרת, מתאים לחבילת המוצרים).
2. **שכבת מעקב: AfterShip** - למעקב אחיד (דואר ישראל + כולם) לדף "איפה ההזמנה שלי".
3. HFD כחלופה/תוספת חזקה (חברת B2C דומיננטית).

## פתוח לאימות (מייל/שיחה קצרה עם כל ספק)
- תנאי גישה ל-API: עלות, sandbox, האם יש חשבון מאסטר/ריסלר (מודל א'), endpoints
  לתוויות + נקודות איסוף + מעקב, הצעת מחיר.
- האם LionWheel/HFD מאפשרים מודל שותף/ריסלר כך שסוחרים שולחים תחת סיאנגו בלי
  התקשרות נפרדת.
