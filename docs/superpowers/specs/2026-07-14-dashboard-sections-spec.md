## עיקרון עיצובי לכלל הדשבורד

כל אזור בדשבורד צריך להרגיש **חי, צבעוני, ומלא-אנרגיה** — לא דף טבלה יבש. כותרת כל אזור מקבלת רקע צבעוני (gradient קל), אייקון גדול וברור, תיאור קצר שמסביר למה זה שווה. Empty states הם הזדמנות — לא טקסט אפור אלא קריאה לפעולה עם איור/אייקון גדול וכפתור בולט.

## 1. ניהול מוצרים / פריטים

### כותרת ומשמעות לפי סוג עסק

| סוג עסק | כותרת | תיאור קצר |
|---|---|---|
| products | המוצרים שלך | כל מה שהחנות שלך מוכרת |
| services | השירותים שלך | מה אתה מציע ללקוחות |
| realestate | הנכסים שלך | נכסים לקנייה, שכירות, ייזום |
| vacation | החדרים והיחידות | מה האורחים יכולים לבחור |
| nonprofit | הפרויקטים שלך | יעדי גיוס ופרויקטים פעילים |
| synagogue | הפרויקטים שלך | יעדי גיוס ופרויקטים פעילים |

### ממשק הרשימה

- **כרטיסים** בגריד (לא שורות טבלה) — כל כרטיס: תמונה ראשית, שם, מחיר/מידע, קטגוריה, אינדיקטור "פעיל / מוסתר".
- **פס עליון:** כפתור "הוסף [מוצר/שירות/נכס/חדר/פרויקט]" + חיפוש + פילטר קטגוריה + toggle גריד/רשימה.
- **מצב ריק (Empty State):** איור גדול + "עדיין אין מוצרים. הוסף את הראשון עכשיו!" + כפתור ראשי. לחנויות שנוצרו עם מוצרי דמו — כרטיס קל בראש: "אלו מוצרי הדגמה, ערוך אותם לשלך".
- **גרור לסידור מחדש** — ידית drag בכרטיס; שמירה אוטומטית.

### טופס הוספה/עריכה

- **Step 1 — מידע בסיסי:** שם (required), מחיר (required), תיאור (textarea + כפתור AI).
- **Step 2 — תמונות:** העלאה / URL / יצירת AI — כרטיסי המקורות בצד לצד (לא dropdown).
- **Step 3 — פרטים נוספים:** קטגוריה, מק"ט, שדות מותאמים-אישית (key/value).
- **Step 4 (אחרי שמירה ראשונה):** וריאנטים (צבעים/מידות/מלאי) — `ProductVariantsEditor`.
- **לוגיקה:** ה-step indicator נשאר בראש; לחיצה על step כלשהו קופצת ישירות אליו.

### הסתרות לפי סוג עסק

- vacation: לא מציג מחיר גלובלי בשדה — מחיר ייקבע לפי חדר (ראה סעיף vacation).
- nonprofit/synagogue: שדה "מחיר" הופך ל-"יעד גיוס (₪)".

## 2. הזמנות / לידים / תרומות / הזמנות לינה

### כותרת ומשמעות לפי סוג עסק

| סוג עסק | כותרת | תיאור קצר |
|---|---|---|
| products | הזמנות | מה הלקוחות הזמינו |
| services | לידים | מי פנה, מה רצה |
| realestate | לידים | מתעניינים בנכסים |
| vacation | הזמנות לינה | מי הזמין, אילו תאריכים |
| nonprofit | תרומות | תרומות שהתקבלו |
| synagogue | תרומות | תרומות שהתקבלו |

### ממשק הרשימה

- **כרטיסים** עם מידע עיקרי ברגל: שם לקוח, פרטי קשר, תאריך, סכום, **chip סטטוס** (צבעוני — ירוק=הושלם, כחול=חדש, אפור=בוטל).
- **פס עליון:** חיפוש לפי שם/טלפון + פילטר סטטוס + מיון לפי תאריך.
- **מצב ריק:** "עדיין לא הגיעו הזמנות. שתף את האתר ותתחיל לקבל!" + כפתור שיתוף.
- **פס צד צבעוני** לפי סטטוס — קל לסריקה.

### מסך פרטי הזמנה

- כרטיס גדול: פרטי לקוח (שם, טלפון, מייל), פירוט פריטים, סכום כולל, הערות, תאריך.
- **שינוי סטטוס:** כפתורים צבעוניים ברורים (לא dropdown).
- vacation מוסיף: תאריכי צ'ק-אין/צ'ק-אאוט, מספר אורחים, שם יחידה.

### סטטוסים לפי סוג עסק

| סוג עסק | סטטוסים |
|---|---|
| products | חדשה → בטיפול → הושלמה / בוטלה |
| services | ליד חדש → בטיפול → נסגר (הצלחה) / לא רלוונטי |
| realestate | ליד חדש → בטיפול → נסגר / לא רלוונטי |
| vacation | ממתין לאישור → מאושר → הגיע → עזב / בוטל |
| nonprofit/synagogue | התקבלה → בבדיקה → נקלטה / בוטלה |

## 3. CRM — לקוחות, ספקים, רווחיות

### מיקום בניווט

CRM הוא **טאב עצמאי** בצד שמאל (לפי הספק שנבחר בסשן הקודם: גישה A — לקוחות+הזמנות בפנים, לא נפרדים). כלומר: כפתור "👥 לקוחות" בניווט הצד → מסך CRM עם 3 טאבים.

### טאב לקוחות

- **כרטיס לקוח:** שם, טלפון, מייל, ספירת הזמנות, סכום כולל, תאריך הזמנה ראשונה.
- **חיפוש ומיון:** לפי שם / מספר הזמנות / סכום כולל.
- **לחיצה על לקוח:** פאנל צד (slider panel) עם פרטים מלאים + היסטוריית הזמנות.
- **Empty State:** "עדיין אין לקוחות. כשיגיעו הזמנות, הם יופיעו כאן."

### טאב ספקים

- רשימת ספקים (שם, איש קשר, טלפון, קטגוריה, הערות).
- הוספה/עריכה פשוטה (לא מורכב).

### טאב רווחיות

- **כרטיסי סיכום:** הכנסות vs. עלויות (ספקים) → רווח נקי.
- **גרף חודשי** (sparkline פשוט — הכנסות/עלויות חודש-חודש).
- תצוגה זו מוגשת כתוספת; מי שאין לו ציין שהנתון חסר.

### Upsell

- בסיס: לקוחות + היסטוריה גלויים לכולם.
- תוספת: רווחיות + ספקים + analytics עמוקים — overlay ברור עם "שדרג לתוספת ה-CRM".
- **ה-overlay אינו חוסם** — הכרטיסים נראים מוצלים ברקע, לחיצה עליהם פותחת את ה-upsell.

## 9. vacation — סוג עסק עצמאי (אירוח)

### רציונל

vacation כיום הוא sub-type תחת `services`, הממופה ל-`realestate`. זה שגוי:
- realestate = מכירה/שכירות נכסים (listings)
- vacation = לינה קצרת-טווח (booking)

vacation צריך להיות `BusinessType` עצמאי עם מודולים ייעודיים.

### זהות העסק

```
BusinessType: "vacation"
תוויות בממשק: "אירוח" / "צימרים ונופש"
אייקון: 🏡
```

**מה vacation כולל:**
- **חדרים ויחידות** (במקום מוצרים)
- **הזמנות לינה** (במקום orders — כולל תאריכי כניסה/יציאה)
- **אורחים** (במקום לקוחות/CRM)
- **מדיניות אירוח** (כללי ביטול, שעות)
- **יומן זמינות** (חדש — ראה מטה)

### ניווט הדשבורד (vacation)

```
📊 סקירה
🛏️ חדרים ויחידות         ← במקום "מוצרים"
📅 הזמנות לינה            ← במקום "הזמנות"
👨‍👩‍👧 אורחים                 ← במקום "לקוחות" (תחת CRM)
📅 יומן זמינות             ← אזור חדש
✍️ תוכן
🎨 עיצוב
📣 שיווק
💳 סליקה
⚙️ הגדרות
✨ תוספות שוות
```

### מסך "חדרים ויחידות"

פריט חדר/יחידה כולל:
- **שם** (חדר דלקס, בית האחד, הקוטג')
- **תיאור** עשיר
- **תמונות** (גלריה, עם כפתור AI)
- **תפוסה מקסימלית** (מספר אורחים)
- **מחיר ללילה** (₪) — לא "מחיר" גנרי
- **מחיר ל-weekend** (₪, אופציונלי)
- **מינימום לילות** (אופציונלי, ברירת מחדל 1)
- **מאפיינים** (שדות מותאמים: Jacuzzi, מטבח, חניה, WiFi...) — checkboxes מוכנות מראש + שדה חופשי
- **מצב:** זמין / לא זמין עכשיו

### מסך "הזמנות לינה"

כרטיס הזמנה מציג:
- שם האורח + טלפון
- **שם היחידה** שהוזמנה
- **תאריכי כניסה / יציאה** + מספר לילות
- **מספר אורחים**
- **סכום כולל**
- **סטטוס:** ממתין לאישור / מאושר / הגיע / עזב / בוטל

CTA מהיר בכרטיס: "אשר" / "בטל" בלי לפתוח פרטים.

### מסך "יומן זמינות" (חדש)

- **תצוגת לוח שנה חודשי** — כל יחידה בשורה נפרדת (multi-resource calendar).
- תאים ירוקים = זמין, אדומים = מוזמן/חסום.
- **לחיצה על תא ריק** → הוסף חסימה ידנית (למשל "שמורה למשפחה").
- **לחיצה על הזמנה קיימת** → פאנל פרטים מהיר.
- זהו **View בלבד בשלב 1** — אפשרות להוספת/עריכת זמינות ידנית בשלב 2.

### "אורחים" (CRM vacation)

- רשימת אורחים עם: שם, טלפון, ספירת ביקורים, תאריך ביקור אחרון.
- לחיצה → היסטוריית ביקורים + יחידות שהם הזמינו.
- לא כולל "ספקים" — לא רלוונטי לאירוח.
- "רווחיות" — רלוונטי ומאפשר לראות הכנסות חודשיות לפי יחידה.

### "מדיניות אירוח" (תחת הגדרות)

- **שעת check-in** (ברירת מחדל: 15:00)
- **שעת check-out** (ברירת מחדל: 11:00)
- **מדיניות ביטול:** שדה טקסט חופשי + בחירה מוכנה מראש (גמיש / מחמיר / ללא החזר)
- **חיות מחמד:** כן / לא / תוספת תשלום
- **עישון:** אסור / מותר בחוץ / מותר
