# אפיון: חנות המודולים - להפוך "וורטיקל" מכלוב לבחירה

> מסמך אפיון לדניאל. לא נגעתי בקוד - זה אפיון בלבד.
> מבוסס על מחקר בקוד: `src/lib/businessModules.ts`, `src/components/onboarding/StepBusinessType.tsx`, `StorefrontVertical.tsx`, `DashboardNav.tsx`.

## שורה תחתונה במשפט אחד

**ההקמה לא משתנה** - סוג העסק עדיין קובע דיפולט אוטומטי (צימר→יומן, בית כנסת→תרומות). מה שמתווסף הוא אזור **"המודולים שלי" בדשבורד** שבו הבעל-עסק מכבה/מדליק/מוסיף יכולות. ה-nav והחנות כבר מרונדרים לפי המודולים הפעילים - זה קיים.

> **החלטת מוצר (מוטי, 13.7):** בשלב הראשון **לא** מציגים ללקוח בורר מודולים בהקמה - זה מסבך. ההקמה נשארת קצרה: בוחר סוג עסק ← מקבל את הפריסט אוטומטית. כל השליטה (כיבוי/הוספה) חיה **בדשבורד בלבד**.

---

## 1. הבעיה - למה הכל מרגיש זהה

המיפוי הנוכחי בקוד:

| סוג עסק (דוגמה) | ממופה ל-`business_type` | מודולים בפועל | הקמה | הבעיה |
|---|---|---|---|---|
| **מאפייה** | `products` | `commerce` | "הוסיפו מוצרים" | תקין |
| **מספרת לק** | `services` | `booking` + `commerce` | "הוסיפו מוצרים" | הקמה זהה למאפייה - לא מגדירים שירותים/שעות, ולכן היומן ריק ("אין משבצות") |
| **צימר** | `realestate` | `listings` | "הוסיפו מוצרים" | אין יומן בכלל, ואין מודול "חדרים/יחידות" - הצימר משתמש ב"נכסים/מוצרים" גנרי |
| **עמותה** | `nonprofit` | `donations` | "הוסיפו מוצרים" | ההקמה עדיין מדברת "מוצרים" |

**שתי תקלות שורש (מבניות, לא ויזואליות):**

1. **ההקמה אחידה.** כל הסוגים עוברים את אותו שלב "הוסיפו מוצרים", גם כשזה חסר משמעות (צימר, מספרה, עמותה). לכן מספרה מסיימת הקמה בלי שירותים ושעות ← היומן שלה ריק.
2. **המודולים דקים / חסרים.** אין מודול אמיתי ל"חדרים/יחידות אירוח", אין "טופס לידים" עצמאי, אין "גלריה/תיק עבודות". צימר נאלץ להשתמש ב"נכסים" גנרי.

## 2. הפתרון - מודל חנות המודולים

העסק מרכיב את האתר ממבחר מודולים. כל מודול = יכולת עצמאית עם מסך ניהול בדשבורד ותצוגה בחנות. הדלקה/כיבוי בקליק.

**הזרימה:**

```
הקמה · שלב 1            הקמה · אוטומטי                    בדשבורד · "המודולים שלי"
בוחר סוג עסק      ←     מקבל את הפריסט לפי הסוג      ←    מכבה / מדליק / מוסיף
                        (צימר→יומן, בית כנסת→תרומות)       יכולות מתי שרוצה
                        בלי בורר, בלי לסבך
```

בעל צימר שרוצה רק לידים ← נכנס לדשבורד, מכבה "יומן" ומדליק "טופס לידים".
צימר אחר שרוצה הזמנות ← מדליק "חדרים + זמינות".
**ההקמה נשארת פשוטה; ההרכבה קורית בדשבורד.**

### מוקאפ א' - אזור "המודולים שלי" (בדשבורד)

```
┌─────────────────────────────────────────────────────────┐
│  דשבורד → המודולים שלי                                    │
├─────────────────────────────────────────────────────────┤
│  🛒 חנות מוצרים            [●] דלוק                        │
│     קטלוג, עגלה ותשלום                                    │
│  📅 יומן ותורים            [●] דלוק                        │
│     שירותים, שעות, יומן חודשי מסונכרן Google              │
│  🏨 חדרים / יחידות אירוח   [ ] כבוי      (חדש)            │
│     יחידות עם זמינות בלוח, מחיר ללילה, הזמנה              │
│  📝 טופס לידים             [ ] כבוי      (חדש)            │
│     "השאירו פרטים" → נכנס ל-CRM עם תזכורות מעקב           │
│  🏠 לוח נכסים              [ ] כבוי                        │
│  🖼️ גלריה / תיק עבודות     [ ] כבוי      (חדש)            │
│  💚 תרומות                 [ ] כבוי                        │
│  🕍 בית כנסת               [ ] כבוי                        │
└─────────────────────────────────────────────────────────┘
```

כל מודול שמדליקים ← הטאב שלו מופיע ב-nav ובחנות אוטומטית (בדיוק כמו היום, רק שהבחירה בידי המשתמש). מודולים בליבה חינם; חלק אפשר לתמחר כתוסף - מתחבר ל"תוספות וכלים" שכבר בנוי.

> **ההקמה נשארת פשוטה - בלי בורר מודולים.** סוג העסק מדליק אוטומטית את הפריסט. היחיד שכן משתנה: **שלב ההגדרה יתאים לפריסט** - צימר יתבקש "הוסיפו יחידה" במקום "הוסיפו מוצרים", מספרה תתבקש "הגדירו שירותים ושעות". כל השליטה במודולים קורית אחר כך בדשבורד.

## 3. ערכות-פתיחה (Presets) לכל סוג עסק

סוג העסק בהקמה נשאר - אבל הוא רק **מדליק אוטומטית** את המודולים ההגיוניים. הכל ניתן לשינוי בדשבורד.

| סוג עסק | מודולים דלוקים כברירת מחדל | שלב ההגדרה שמתאים |
|---|---|---|
| מאפייה / חנות | מוצרים | הוספת מוצרים |
| מספרה / קליניקה / לק | יומן ותורים + מוצרים | הגדרת שירותים + שעות פעילות |
| צימר / נופש | חדרים/יחידות + גלריה + לידים | הוספת יחידה + תמונות |
| מתווך / רכב | לוח נכסים + לידים | הוספת נכס ראשון |
| צלם / מעצב / בעל מקצוע | גלריה + לידים + יומן | העלאת עבודות |
| עמותה | תרומות | יצירת קמפיין |
| בית כנסת | תרומות + בית כנסת | הגדרת עליות/מקומות |

## 4. קטלוג המודולים - קיים מול חדש

| מודול | סטטוס בקוד | מה חסר כדי לסגור |
|---|---|---|
| חנות מוצרים (`commerce`) | קיים ומלא | - |
| יומן ותורים (`booking`) | קיים | לוודא שההקמה מגדירה שירותים+שעות (אחרת יומן ריק) |
| לוח נכסים (`listings`) | קיים | - |
| תרומות (`donations`) | קיים | - |
| בית כנסת (`synagogue`) | קיים | - |
| חדרים / יחידות אירוח | **חדש** | מודול חדש: יחידה (שם/מחיר-ללילה/תמונות) + זמינות בלוח + הזמנה. אפשר לבנות מעל תשתית ה-`booking`. |
| טופס לידים עצמאי | **חדש (חלקי)** | הלוגיקה קיימת ב-`listings` (פנייה→CRM); לחלץ ל"מודול לידים" שאפשר להדליק לבד בלי לוח נכסים. |
| גלריה / תיק עבודות | **חדש** | מודול תצוגה: פריטי גלריה עם תמונה+כותרת+קטגוריה. פשוט יחסית. |

---

## 5. למה זה שומר על המבנה של דניאל

- **התשתית כבר מודולרית.** `getEnabledModules(business)` כבר מחזיר מערך מודולים, וה-nav (`DashboardNav`) והחנות (`StorefrontVertical`) כבר מרונדרים לפיו. אנחנו **לא** משנים ארכיטקטורה.
- **השינוי המרכזי הוא מקור הבחירה:** במקום `DEFAULT_MODULES[type]` קשיח ← עמודה `enabled_modules` על העסק (מאותחלת מה-preset, ניתנת לעריכה). `getEnabledModules` קורא ממנה.
- **ההקמה לא משתנה מבחינת הלקוח** - אין בורר מודולים. הפריסט מוחל אוטומטית. היחיד שמשתנה: שלב ההגדרה יתאים לפריסט (למספרה - שירותים+שעות; לצימר - יחידה). שאר ההקמה זהה.
- **אזור "המודולים שלי"** = עמוד חדש אחד **בדשבורד** (כמו "תוספות וכלים" שכבר קיים) שמדליק/מכבה רשומות ב-`enabled_modules`. נקי.

## 6. הצעת שלבים (כדי לא לעשות הכל בבת אחת)

1. **שלב 1 - התשתית (הכי משפיע):** עמודת `enabled_modules` (מאותחלת מהפריסט האוטומטי) + אזור "המודולים שלי" **בדשבורד** - הדלקה/כיבוי/הוספה של מה שכבר קיים (`commerce`/`booking`/`listings`/`donations`). כבר נותן ערך אדיר בלי מודולים חדשים ובלי לגעת בהקמה.
2. **שלב 2 - שלב ההגדרה בהקמה:** להתאים את שלב ההגדרה לפריסט (מספרה - שירותים+שעות) - סוגר את באג "היומן ריק". **עדיין בלי בורר מודולים ללקוח.**
3. **שלב 3 - מודולים חדשים:** חדרים/יחידות (לצימר) → גלריה → טופס-לידים עצמאי.
